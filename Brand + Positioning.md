## 3. MVP Product Spec (so this can become a real platform)

This is the ‚Äúwhat to build‚Äù list you can hand to Cline / a dev.

### Core user types

1. **Borrower** ‚Äì business applying for credit
2. **Investor** ‚Äì accredited investor reviewing/funding deals
3. **Admin / Credit team** ‚Äì you + whoever underwrites

### Borrower flow (v1)

1. **Signup & KYC/KYB**

   * Email + password (or magic link)
   * Company legal name, EIN, address
   * Owner info

2. **Application**

   * Amount requested, purpose, use of funds
   * Revenue, EBITDA, existing debt
   * Upload last 12‚Äì24 months P&L + balance sheet (PDFs)

3. **Bank connection (later)**

   * Plaid connect to pull transaction history

4. **Status page**

   * Shows ‚ÄúUnder review / Term sheet / In funding / Funded‚Äù
   * Messaging area to ask questions

### Investor flow (v1)

1. **Request access**

   * Simple form: name, email, firm, AUM, check size, accreditation checkbox

2. **Back-office approval**

   * You manually review and ‚Äúapprove‚Äù in admin panel

3. **Deal room**

   * Once approved, investors can see:

     * Deal name
     * Requested amount
     * Target yield / structure (e.g., 12% interest, 3-year term)
     * Downloadable credit memo and financials
   * ‚ÄúIndicate interest‚Äù button (no actual wiring yet in v1 ‚Äî you can handle offline)

### Admin / Credit tools

* Dashboard of all applications + status
* Internal notes
* A simple ‚ÄúGenerate credit memo‚Äù button that takes the form data + financials and outputs a formatted PDF (you can use AI later for this)

### Data model (simple starting point)

Tables (or Supabase/Postgres schema):

* `users` (id, role: borrower/investor/admin, email, name)
* `companies` (id, name, industry, revenue, ebitda, owner_user_id)
* `applications` (id, company_id, amount_requested, purpose, status, created_at)
* `documents` (id, application_id, type, file_url)
* `investors` (id, user_id, firm_name, check_size_min, check_size_max, accredited_bool)
* `investor_interest` (id, investor_id, application_id, amount_indicated, status)

That‚Äôs enough to get a functioning private beta.
